#### Workflow : Attrition Advisor

| Workflow Name | Attrition Advisor |
|---------------|---------------|
| **Code** | `XX_ATTRITION_ADVISOR` |
| **Description** | Handles natural-language attrition questions by invoking the Smart Answers Expert workflow, validating the returned result, and formatting the response for display. |
| **Expose to Agentic Apps** | On |
| **Input Parameters** | `message` |
| **Output Parameters** | 1. `Success`<br>&nbsp;&nbsp;&nbsp;1. Uses `multiRecordWidget` rendering for the final result display<br>&nbsp;&nbsp;&nbsp;2. Uses `Smart Answers Expert` workflow and `HR Attrition Analytic Reports` BO<br>&nbsp;&nbsp;&nbsp;3. Uses the downstream Smart Answers output after validation/sanitation<br>&nbsp;&nbsp;&nbsp;4. Displays attrition query results for the logged-in user's accessible data scope, along with a generated insight summary above the records<br>2. `Fallback`<br>&nbsp;&nbsp;&nbsp;1. Triggered when the downstream workflow output is not usable or the query is too vague<br>&nbsp;&nbsp;&nbsp;2. Uses `Smart Answers Expert` workflow<br>&nbsp;&nbsp;&nbsp;3. Prompts the user to provide a more detailed attrition question<br>&nbsp;&nbsp;&nbsp;4. Guides the user toward supported filters such as action reason, termination date, location, job, department, business unit, and manager |

#### Workflow : Top Leavers by Job

| Workflow Name | Top Leavers by Job |
|---------------|---------------|
| **Code** | `XX_TOP_LEAVERS_BY_JOB` |
| **Description** | Shows attrition across the organization by grouping leavers by job role, enabling job and manager-level drilldown, suggesting active employees with similar attrition risk patterns, and surfacing corrective actions alongside those suggestions. |
| **Expose to Agentic Apps** | On |
| **Input Parameters** | `JobName` |
| **Output Parameters** | 1. `InitDisplay`<br>&nbsp;&nbsp;&nbsp;1. Uses chart layout / pie chart rendering<br>&nbsp;&nbsp;&nbsp;2. Uses `HR Attrition Analytic Reports` BO<br>&nbsp;&nbsp;&nbsp;3. Displays the top 5 jobs with the highest leaver count for the organizational scope in the workflow<br>&nbsp;&nbsp;&nbsp;4. Displays job names, leaver counts, and a short executive summary of concentration/outlier patterns<br>2. `AdditionalContent`<br>&nbsp;&nbsp;&nbsp;1. Uses `messageListWidget`<br>&nbsp;&nbsp;&nbsp;2. Uses `HR Attrition Analytic Reports` BO<br>&nbsp;&nbsp;&nbsp;3. Displays the full sorted job list for leavers beyond the chart-focused top slice view<br>&nbsp;&nbsp;&nbsp;4. Displays each job name and leaver count as badge text with row action logic `ora.Invoke("rowNavigate", {"command":"rowNavigate","data":{"job": $Job, "count": $EmpCount}})`<br>3. `InitActions`<br>&nbsp;&nbsp;&nbsp;1. Uses `oraInsight`-style action rendering<br>&nbsp;&nbsp;&nbsp;2. Uses `HR Attrition Analytic Reports` BO<br>&nbsp;&nbsp;&nbsp;3. Displays a high-attrition alert for the top job when its count is materially higher than the second-highest job<br>&nbsp;&nbsp;&nbsp;4. Displays action logic `ora.Invoke("rowNavigate", {"command":"rowNavigate","data":{"job": $jobName, "count": $EmpCount}})` with action text `View Trend`<br>4. `FillParameters`<br>&nbsp;&nbsp;&nbsp;1. Uses communication parameter filling for downstream share/export flows<br>&nbsp;&nbsp;&nbsp;2. Uses `HR Attrition Analytic Reports` BO<br>&nbsp;&nbsp;&nbsp;3. Prepares the output payload for communication/export use<br>&nbsp;&nbsp;&nbsp;4. Includes serialized job attrition data and PDF title metadata<br>5. `Default`<br>&nbsp;&nbsp;&nbsp;1. Returns plain JSON output<br>&nbsp;&nbsp;&nbsp;2. Uses `HR Attrition Analytic Reports` BO<br>&nbsp;&nbsp;&nbsp;3. Displays a machine-readable job/count payload for integrations or default consumers<br>&nbsp;&nbsp;&nbsp;4. Returns items in the form `{"items":[{"job":"...","count":number}]}`<br>6. `InvokeAction`<br>&nbsp;&nbsp;&nbsp;1. `displayEmp`<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1. Uses two `multiRecordWidget` outputs plus `oraInsight` actions for `Promote` and `Change Salary`<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;2. Uses `HR Attrition Analytic Reports`, `HR Attrition Personal Searches`, and `HR Attrition Action Tool`<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;3. Displays the selected leaver details for that manager/job combination and highlights active employees with high attrition scores as potential leavers<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;4. Displays attrition score as a badge in the potential leavers details and surfaces `Promote` / `Change Salary` quick actions from generated links<br>&nbsp;&nbsp;&nbsp;2. `rowNavigate`<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1. Uses multirecord/list rendering for manager drilldown<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;2. Uses `HR Attrition Analytic Reports` BO<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;3. Displays manager-wise breakdown for the selected job<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;4. Uses row action logic `ora.Invoke("displayEmp", {"command":"displayEmp","data":{"jobName": $JobName, "count": $EmpCount, "managerPersonId": $ManagerPersonId, "managerAssignmentId": $ManagerAssignmentId, "managerName": $ManagerName, "managerDepartmentName": $ManagerDepartmentName, "managerCount": $ManagerEmpCount}})` for manager drilldown rows<br>&nbsp;&nbsp;&nbsp;3. `INVALID_ACTION`<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1. Uses return/error output only<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;2. Uses action payload validation logic<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;3. Displays an action validation failure state<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;4. Displays the validation error message from the action payload |
