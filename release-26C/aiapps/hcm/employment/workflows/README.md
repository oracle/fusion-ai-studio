# Employment Workflows

#### Workflow : Attrition Advisor

| Workflow Name | Attrition Advisor |
|---------------|---------------|
| **Code** | `XX_ATTRITION_ADVISOR` |
| **Description** | Handles natural-language attrition questions by invoking the Smart Answers Expert workflow, validating the returned result, and formatting the response for display. |
| **Expose to Agentic Apps** | On |
| **Input Parameters** | `message` |
| **Output Parameters** | This workflow uses the multiRecordWidget to render the final results. It leverages the Smart Answers Expert workflow together with the HR Attrition Analytic Reports BO and displays the downstream Smart Answers output only after it has been validated and sanitized. The results are limited to the logged-in user’s accessible data scope and include a generated insight summary displayed above the returned records. |

#### Workflow : Smart Answers Expert

| Workflow Name | Smart Answers Expert |
|---------------|---------------|
| **Code** | `XX_SMARTANSWERSEXPERT` |
| **Description** | Converts a user's attrition analysis request into a safe OTBI logical SQL query, executes it, and returns the result data with metadata for downstream workflows. |
| **Expose to Agentic Apps** | On |
| **Input Parameters** | `message` |
| **Output Parameters** | The workflow uses OTBI logical SQL generated from the user’s question and metadata rules, together with the HR Attrition Analytic Reports BO, to retrieve raw attrition query results that serve as the execution dataset for downstream workflows and display handlers. It also generates a text-based response that interprets the user’s attrition question, which can be consumed directly by parent workflows such as Attrition Advisor. In addition, the workflow returns execution status information, including job and workflow status, enabling parent workflows to detect success or failure and monitor long-running or multi-step executions. Alongside the functional output, it produces workflow metadata—such as workflow code, workflow version, token counts, timestamps, and execution timing—to help downstream workflows validate, interpret, and reason about the returned payload.|

#### Workflow : Top Leavers by Job

| Workflow Name | Top Leavers by Job |
|---------------|---------------|
| **Code** | `XX_TOP_LEAVERS_BY_JOB` |
| **Description** | Shows attrition across the organization by grouping leavers by job role, enabling job and manager-level drilldown, suggesting active employees with similar attrition risk patterns, and surfacing corrective actions alongside those suggestions. |
| **Expose to Agentic Apps** | On |
| **Input Parameters** | `JobName` |
| **Output Parameters** | InitDisplay: The InitDisplay view uses a chart layout with pie chart rendering to visualize the top five jobs with the highest leaver counts for the organizational scope defined in the workflow. It leverages the HR Attrition Analytic Reports BO to retrieve the data and presents job names, corresponding leaver counts, and a concise executive summary highlighting concentration trends and notable outlier patterns.<br/>AdditionalContent: The AdditionalContent view uses the messageListWidget to display the complete, sorted list of jobs by leaver count beyond the top five shown in the chart. Using the HR Attrition Analytic Reports BO, it presents each job name and its leaver count as badge text, with row-level navigation enabled through the ora.Invoke("rowNavigate", ...) action for further drill-down.<br/>InitActions: The InitActions view uses oraInsight-style action rendering to surface proactive insights. Based on data from the HR Attrition Analytic Reports BO, it highlights a high-attrition alert when the leading job’s leaver count is significantly higher than the second-highest job and provides a View Trend action that invokes row navigation to explore the underlying data.<br/>FillParameters: The FillParameters view prepares communication parameters for downstream sharing and export scenarios. Using the HR Attrition Analytic Reports BO, it generates a payload containing serialized job attrition data along with PDF title metadata to support communication and reporting workflows.<br/>Default: The Default view returns a machine-readable JSON payload using the HR Attrition Analytic Reports BO for integrations and other default consumers. The response contains a list of job names and their corresponding leaver counts in the format {"items":[{"job":"...","count":number}]}.<br/>InvokeAction: The InvokeAction flow supports two interactive drill-down actions. The displayEmp action uses two multiRecordWidget views together with oraInsight actions to display leaver details for the selected manager and job combination while also identifying active employees with high attrition scores as potential future leavers. It leverages the HR Attrition Analytic Reports, HR Attrition Personal Searches, and HR Attrition Action Tool BOs, displaying attrition scores as badges and surfacing quick actions such as Promote and Change Salary. The rowNavigate action uses multi-record and list rendering to present a manager-wise breakdown for the selected job using the HR Attrition Analytic Reports BO. Each manager row supports further drill-down by invoking the displayEmp action with the relevant manager and job context.|

#### Workflow : Top Leavers By Manager

| Workflow Name | Top Leavers By Manager |
|---------------|---------------|
| **Code** | `XX_TOP_LEAVERS_BY_MANAGER` |
| **Description** | Shows attrition across the logged-in employee’s reporting hierarchy by grouping leavers under each manager, enabling manager-level drilldown, suggesting active employees with similar attrition risk patterns, and surfacing corrective actions alongside those suggestions. |
| **Expose to Agentic Apps** | On |
| **Input Parameters** | `ManagerId` |
| **Output Parameters** | InitDisplay: The InitDisplay view uses message-list widget rendering to show the top five managers with the highest leaver counts within the logged-in user’s reporting hierarchy scope. It relies on the HR Attrition Analytic Reports BO and presents each manager’s name, department subtitle, and count badge, along with an executive summary and row-level drill-down through ora.Invoke("rowNavigate", {"command":"rowNavigate","data":{"managerId": $ManagerId, "count": $EmpCount}}).<br/>AdditionalContent: The AdditionalContent view uses message-list widget rendering to present the remaining managers beyond the top five shown in the initial view. It also uses the HR Attrition Analytic Reports BO and displays each manager’s name, department, leaver count, and clickable row output through ora.Invoke("rowNavigate", {"command":"rowNavigate","data":{"managerId": $ManagerId, "count": $EmpCount}}).<br/>InitActions: The InitActions view uses oraInsight-style action rendering to surface proactive insights. Based on the HR Attrition Analytic Reports BO, it raises a high-attrition alert when the top manager’s count is at least 1.5 times the second-highest manager’s count and provides a View Trend action that invokes row navigation using ora.Invoke("rowNavigate", {"command":"rowNavigate","data":{"managerId": $ManagerId, "count": $EmpCount}}).<br/>FillParameters: The FillParameters view prepares communication parameters for downstream sharing and export scenarios. Using the HR Attrition Analytic Reports BO, it packages the manager attrition data into a serialized payload for communication and export use.<br/>Default: The default view returns the raw result output using the HR Attrition Analytic Reports BO. It produces a machine-consumable manager results payload and returns the full manager result set for default consumers.<br/>InvokeAction: The InvokeAction flow supports two drill-down actions. The displayEmp action uses two multiRecordWidget outputs together with oraInsight actions for Promote and Change Salary. It leverages the HR Attrition Analytic Reports, HR Attrition Personal Searches, and HR Attrition Action Tool BOs to display selected leaver details for the manager and highlight active employees with high attrition scores as potential leavers, with the attrition score shown as a badge and quick actions surfaced from generated links. The rowNavigate action uses multirecord employee detail rendering with the HR Attrition Analytic Reports BO to display leaver employee details for the selected manager, and it invokes ora.Invoke("displayEmp", {"command":"displayEmp","data":{"name": $EmployeeName, "department": $Department, "hireDate": $HireDate, "terminationDate": $TerminationDate, "location": $Location, "actionReason": $ActionReason, "Quartile": $Quartile, "jobName": $JobName, "managerId": $ManagerId, "managerAssignmentId": $ManagerAssignmentId}}) for employee-detail drill-down rows.|

#### Workflow : Top Leavers by Reason

| Workflow Name | Top Leavers by Reason |
|---------------|---------------|
| **Code** | `XX_TOP_LEAVERS_BY_REASON` |
| **Description** | Shows attrition across the logged-in employee’s reporting hierarchy by grouping leavers by reason, enabling reason-level drilldown, surfacing manager-wise and employee-level views, suggesting active employees with similar attrition risk patterns, and presenting corrective actions where applicable. |
| **Expose to Agentic Apps** | On |
| **Input Parameters** | `Reason` |
| **Output Parameters** | initDisplay: The initDisplay view uses message-list widget rendering to show the top five attrition reasons within the logged-in user’s accessible hierarchy scope. It relies on the HR Attrition Analytic Reports BO and presents each reason with a count badge, an executive summary, and row-level drill-down through ora.Invoke("rowNavigate", {"command":"rowNavigate","data":{"reason": $ActionReason, "count": $EmpCount}}).<br/>additionalContent: The additionalContent view uses message-list widget rendering to display the remaining attrition reasons beyond the top five shown in the initial view. It also uses the HR Attrition Analytic Reports BO and shows each reason with its count, along with clickable row navigation through ora.Invoke("rowNavigate", {"command":"rowNavigate","data":{"reason": $ActionReason, "count": $EmpCount}}).<br/>initActions: The initActions view uses oraInsight-style action rendering to surface proactive insights. Based on the HR Attrition Analytic Reports BO, it highlights high-attrition alerts for reasons whose counts are above the average reason count and provides a View Trend action that invokes row navigation through ora.Invoke("rowNavigate", {"command":"rowNavigate","data":{"reason": $ActionReason, "count": $EmpCount}}).<br/>FillParameters: The FillParameters view prepares communication parameters for export and share flows. Using the HR Attrition Analytic Reports BO, it packages the reason-level attrition data into a serialized payload for downstream communication and export use.<br/>default: The default view returns the raw result output using the HR Attrition Analytic Reports BO. It provides a machine-consumable reason result payload and returns the filtered reason results for default consumers.<br/>InvokeAction: The InvokeAction flow supports two drill-down actions. The displayEmp action varies by context: for Compensation, it uses multirecord manager-level rendering with the HR Attrition Analytic Reports BO to show a manager-level compensation attrition breakdown for the selected reason and to support follow-on drill-down; for Career Growth and Others, it uses two multiRecordWidget outputs together with oraInsight actions for Promote and Change Salary, leveraging the HR Attrition Analytic Reports, HR Attrition Personal Searches, and HR Attrition Action Tool BOs to show selected leaver details for the manager/reason combination, highlight active employees with high attrition scores as potential leavers, and surface quick actions from generated links. The rowNavigate action uses multirecord manager-level rendering with the HR Attrition Analytic Reports BO to display a manager-wise breakdown for the selected reason, and it invokes ora.Invoke("displayEmp", {"command":"displayEmp","data":{"reason": $ActionReason, "count": $EmpCount, "managerPersonId": $ManagerPersonId, "managerAssignmentId": $ManagerAssignmentId, "managerName": $ManagerName, "managerDepartmentName": $ManagerDepartmentName, "managerCount": $ManagerEmpCount}}) for manager drill-down rows.|
